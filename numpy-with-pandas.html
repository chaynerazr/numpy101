Pandas and Numpy#

Just want to show this beautiful plot before moving on. The plot actually use numpy’s array and numpy’s random function as well.

from matplotlib import rcParams, cycler
import matplotlib.pyplot as plt
import numpy as np

This plot look familiar?

Yep, it’s the exmaple plot you get when you create Jupyter Book.

# Fixing random state for reproducibility
np.random.seed(19680801)

N = 10
data = [np.logspace(0, 1, 100) + np.random.randn(100) + ii for ii in range(N)]
data = np.array(data).T
cmap = plt.cm.coolwarm
rcParams['axes.prop_cycle'] = cycler(color=cmap(np.linspace(0, 1, N)))


from matplotlib.lines import Line2D
custom_lines = [Line2D([0], [0], color=cmap(0.), lw=4),
                Line2D([0], [0], color=cmap(.5), lw=4),
                Line2D([0], [0], color=cmap(1.), lw=4)]

fig, ax = plt.subplots(figsize=(10, 5))
lines = ax.plot(data)
ax.legend(custom_lines, ['Cold', 'Medium', 'Hot']);

Numpy vs Pandas#

To use pandas, first of all, you need to install it.

conda install pandas

Then you can import it.

import pandas as pd

We also can make numpy multi-dimensional array like doing in former chapter as:

np_array = np.array([[12, 4, 5, 44, 1],
                    [5, 2, 54, 3, 2], 
                    [20, 16, 7, 3, 8],
                    [14, 3, 17, 2, 6]])

And then you can also calculate mean of

np.array([[12, 4, 5, 44, 1],
          [5, 2, 54, 3, 2], 
          [20, 16, 7, 3, 8],
          [14, 3, 17, 2, 6]]).mean(axis=0)

array([12.75,  6.25, 20.75, 13.  ,  4.25])

This follows the mathematic formula (10).

(10)#\[A=\frac{1}{n} \sum_{i=i}^{n} x_{i}\]

or for Sum function that follow this (11).

(11)#\[A= \sum_{i=1}^k x_i \]

And the code for that is

np.array([[12, 4, 5, 44, 1],
          [5, 2, 54, 3, 2], 
          [20, 16, 7, 3, 8],
          [14, 3, 17, 2, 6]]).sum(axis=0)

array([51, 25, 83, 52, 17])

According to this website, Numpy is faster than Panda in any way.

Which one is faster?

Shout out to Eunjoo Byeon who has detail analysis comparing Numpy and Pandas Please visit https://towardsdatascience.com/speed-testing-pandas-vs-numpy-ffbf80070ee7

From Fig. 8 and Fig. 9, Numpy works quicker espeacially when data set become bigger.

So we can use to_numpy method of pandas to convert to numpy array.

df = pd.DataFrame({"A":[12, 4, 5, 44, 1],
                   "B":[5, 2, 54, 3, 2], 
                   "C":[20, 16, 7, 3, 8],
                   "D":[14, 3, 17, 2, 6]})
np_array = df.to_numpy()
np_array

array([[12,  5, 20, 14],
       [ 4,  2, 16,  3],
       [ 5, 54,  7, 17],
       [44,  3,  3,  2],
       [ 1,  2,  8,  6]])

Then we can use regular numpy method.

np_array.mean(axis=1)

array([12.75,  6.25, 20.75, 13.  ,  4.25])

Numpy and Panda are linked#

The amazing thing when you do like that actually linked. When you change the value in the array, the linked dataframe also change.

The following code will change the last column to be the median of number from column 1-3.

np_array[:,3]=np.median(np_array[:,0:2], axis=1)
np_array

array([[12,  5, 20,  8],
       [ 4,  2, 16,  3],
       [ 5, 54,  7, 29],
       [44,  3,  3, 23],
       [ 1,  2,  8,  1]])

Then when we check the values in original dataframe

df
A B C D
0 12 5 20 8
1 4 2 16 3
2 5 54 7 29
3 44 3 3 23
4 1 2 8 1

It gets updated as well.
